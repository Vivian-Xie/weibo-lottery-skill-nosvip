# 如何手动获取微博 Cookie（给用户的图文指引）

> 💡 **首选用 RPA 自动获取**：运行 `scripts/get_cookie_rpa.py` 会弹出浏览器让你登录一次，之后自动读取 cookie，无需 F12 手动复制。本文档是 RPA 不可用（无图形界面等）时的**降级手动方案**。

> AI 无法替你获取 cookie：它是你登录微博后浏览器才持有的私密凭证，AI 不能登录你的账号，也无权读你浏览器的数据。这是平台安全机制（能自动拿到就等于能盗号了）。所以这一步**必须你亲自做一次**，30 秒搞定。cookie 没过期之前可一直复用。

## 步骤（电脑 Chrome / Edge）

1. 用浏览器打开并**登录** https://weibo.com （确保是已登录状态）。
2. 按 **F12** 打开开发者工具（或右键 → 检查）。
3. 切到顶部的 **Network（网络）** 标签页。
4. 按 **F5 / Ctrl+R** 刷新一下页面，让请求列表出现内容。
5. 在请求列表里**随便点一个** `weibo.com` 的请求（比如名字像 `config`、`statuses`、或第一条 `weibo.com` 文档请求都行）。
6. 在右侧面板找到 **Headers（标头）** → 往下拉到 **Request Headers（请求标头）**。
7. 找到 **`Cookie:`** 这一整行，把冒号后面**一整串**复制出来（很长，从 `SCF=...` 或 `SINAGLOBAL=...` 开头，一直到最后）。

## 把 cookie 给 AI

直接把复制的整行 cookie 粘贴到对话里发给 AI 即可。AI 会把它存成 `cookie.txt`。

## 检查是否抓全了

有效的 cookie **必须包含这几个核心字段**（缺了接口会返回空 / 跳登录页）：

- `SUB=...`     ← 最关键，登录态
- `SUBP=...`    ← 登录态签名
- `ALF=...`     ← 过期时间戳

辅助字段（有更好）：`SCF`、`SINAGLOBAL`、`XSRF-TOKEN`、`WBPSESS`、`ULV`。

示例（字段顺序无所谓，整行一段即可）：

```
SCF=Axxx; SINAGLOBAL=349...; XSRF-TOKEN=CDe5...; SUB=_2A25...; SUBP=0033...; ALF=02_178...; WBPSESS=8K3Q...
```

## Cookie 过期了怎么办

如果某天爬取突然返回空 / 报错，多半是 cookie 过期了（`ALF` 到期）。重复上面步骤重新抓一次、再发给 AI 即可。
